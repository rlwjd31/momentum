- Form tag를 통해 submit을 하면 페이지는 새로고침이된다.
- submit은 form안의 button을 클릭하거나 Enter를 하게 되면 submit이 이루어진다. 
- onSubmit과 같은 event function들은 js는 인자를 넣어서 실행시킨다.
    - 아래 preventDefault()는 브라우저가 자동으로 실행하는 동작을 막는다.
    - 1. preventDefault() : submit을 하게되면 브라우저는 새로고침을 하는데 이 새로고침을 방지한다.
		-> 또한 다른 예를 들자면 anchor tag의 click을 하면 다른 페이지로 이동하는데 preventDefault를 쓰면 이를 방지한			다.
- localStorage : 브라우저에 공짜로 뭔가를 기억할 수 있게 해주는 API가 localStorage이다. 즉, 브라우저에 뭔가를 저장할 수 있게 해준다.
  -> setItem(key, value)로 저장하거나 removeItem(key)로 삭제할 수 있다.
  -> 근데 localStorage에 value로 object를 저장하려고 했지만 문자열로 저장해버린다....;; 엥...?
  -> 문자열로 저장하기에는 좋아보임...;;
  -> 하다보니 해결책이 있는데 localStorage에 배열이나, 객체를 저장하기 위해서는 JSON.stringfy()로 parsing한 다음에 넣어주면된다.
  -> 또한 JSON.stringfy()를 통해 생성된 문자열을 원 객체로 돌리기 위해서는 JSON.parse()를 이용하면된다.
  -> 다른 중요한 점은, 새로고침을 하면 js파일안의 변수에 저장된 값들이 null이 되는데 localStorage는 그대로 보존한다.


*** preventDefault *** :  어떤 event의  기본행동(어떤 function에 대해 브라우저가 기본적으로 수행하는 동작[Ex) 새로고침])이든지 발생되지 않도록 막는 함수이다.


- setInterval : 몇 초마다 원하는 것을 실행시킬 때 사용한다.
  -> Ex) function sayHello(){ console.log("hi!!!")}
         setInterval(sayHello, 5000)
         -> 5초마다 sayHello함수를 실행시킨다.

- setTimeout : 몇 초뒤에 원하는 것을 한 번 실행시킬 때 사용한다.
  -> Ex) function sayHello(){ console.log("hi!!!")}
         setTimeout(sayHello, 5000)
         -> 5초뒤에 sayHello함수를 실행시킨다.

- padStart(길이, 앞에 붙일 것) : 
  -> Ex) const str1 = '5';
         console.log(str1.padStart(2, '0'));
         // expected output: "05"
  -> https://developer.mozilla.org/ko/docs/Web/JavaScript/Reference/Global_Objects/String/padStart 

- js random
  -> javascript의 random은 Math library에서 제공하고 있다. Math.random()은 0~1사이의 숫자를 제공하며,
     만약, [0, 9] 숫자를 얻기 위해서는 Math.random()*10을 하면 0 ~ 9사이의 숫자를 얻을 수 있다.


- document.body.appendChild(document.createElement()) : body자식의 맨 마지막에 생성한 요소를 붙인다.
- document.body.prependChild(document.createElement()) : body자식의 맨 앞에 생성한 요소를 붙인다.
